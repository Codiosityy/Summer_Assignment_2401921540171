<details>
<summary><b>📌 Week 1 — Day 1 Solution Summaries</b></summary>

<br/>

**Problem 1: Two Sum** — Find two numbers that add up to a target.
> **Approach**: Hash map stores seen values + indices; look up the complement in O(1).
> **Time**: O(n) | **Space**: O(n)

**Problem 26: Remove Duplicates from Sorted Array** — Remove duplicates in-place, return new length.
> **Approach**: Two pointers — `read` scans, `write` overwrites duplicates.
> **Time**: O(n) | **Space**: O(1)

**Problem 121: Best Time to Buy and Sell Stock** — Max profit from one buy-sell transaction.
> **Approach**: Track min price so far, compute best profit in a single pass.
> **Time**: O(n) | **Space**: O(1)

</details>

<details>
<summary><b>📌 Week 1 — Day 2 Solution Summaries</b></summary>

<br/>

**Problem 53: Maximum Subarray** — Find the contiguous subarray with the largest sum.
> **Approach**: Kadane's algorithm — at each index, either start a new subarray or extend the current best.
> **Time**: O(n) | **Space**: O(1)

**Problem 217: Contains Duplicate** — Return true if any value appears at least twice.
> **Approach**: Hash set tracks seen elements; first repeat returns true.
> **Time**: O(n) | **Space**: O(n)

**Problem 643: Maximum Average Subarray I** — Find the contiguous subarray of size `k` with the highest average.
> **Approach**: Sliding window of size `k`; track sum and slide in O(1) per step.
> **Time**: O(n) | **Space**: O(1)

</details>

### 📚 OOPS Problem Briefs

<details>
<summary><b>Week 1 — LibraryUser Interface</b></summary>

<br/>

**Scenario**: A library application for Adults and Children. Both register accounts; rules differ by age and book type.

- Create interface `LibraryUser` with `registerAccount()` and `requestBook()`
- Implement `KidUser` (age < 12, "Kids" books, 10 days)
- Implement `AdultUser` (age > 12, "Fiction" books, 7 days)
- Test in `LibraryInterfaceDemo.java` with provided test cases

> **Concepts**: Interface, Polymorphism, Encapsulation, Future extensibility

</details>

<details>
<summary><b>Week 2 — Playable Interface (Music)</b></summary>

<br/>

**Scenario**: Implement a music hierarchy using packages.

- Interface `Playable` in package `music` with `void play();`
- Class `Veena` in `music.string` implements `Playable`
- Class `Saxophone` in `music.wind` implements `Playable`
- Test class in package `live` calls `play()` directly and via `Playable` reference

> **Concepts**: Interface, Packages, Polymorphic dispatch

</details>

<details>
<summary><b>Week 3 — Compartment Abstract Class</b></summary>

<br/>

**Scenario**: Abstract `Compartment` representing rail coach; subclasses provide different notices.

- Abstract class `Compartment` with abstract `notice()` method
- Subclasses: `FirstClass`, `Ladies`, `General`, `Luggage` — each overrides `notice()`
- `TestCompartment` declares an array of size 10, randomly instantiates each slot (1-4)
- Demonstrates polymorphic behavior of `notice()`

> **Concepts**: Abstract class, Polymorphism, Random behavior

</details>

<details>
<summary><b>Week 4 — Mixed OOPS (4 Questions)</b></summary>

<br/>

**Q1**: Create interface `Test` with method `square()`. Implement in `Arithmetic` class. Use via `ToTestInt`.
**Q2**: Outer class with `display()`; inner class `Inner` with `display()`. Call both from main.
**Q3**: Class `Point` with private `x, y`, constructors, setters (`setX`, `setY`, `setXY`).
**Q4**: Class `Box` and subclass `Box3D` — constructors, area, volume.

> **Concepts**: Interface, Inner class, Encapsulation, Inheritance, Method overriding

</details>

## 📌 File Naming Convention

| Type | Pattern | Example |
|------|---------|---------|
| **LeetCode Solution (C++)** | `<sequence>.cpp` (1-indexed within each day) | `Day_1/1.cpp`, `Day_2/3.cpp` |
| **OOPS Program (Java)** | `<DescriptiveName>.java` | `LibraryUserDemo.java` |

> 💡 **File numbering**: Solutions are numbered sequentially within each `Day_Y/` folder (`1.cpp`, `2.cpp`, `3.cpp`). The LeetCode problem number is shown in the Solutions Index table, not the filename — this keeps filenames short and ordered.
>
> 💡 **Why `.cpp` for DSA, `.java` for OOPS?** — LeetCode is C++ here, but the OOPS assignment specifies Java. Each language lives in its own convention.

### ➕ Adding a New Solution (Quick Guide)

1. **Pick the right folder** → `Week_X/Day_Y/` for DSA, `Week_X/OOPS/` for OOPS
2. **Name the file** → Use the next sequential number in that day: `<n>.cpp` (or descriptive `.java`)
3. **Write clean code** → Use the standard `Solution` class, add comments for approach & complexity
4. **Update this README** → Flip the row from 🚧 Pending to ✅ Done, link the file
5. **Commit with a clear message** → e.g., `Add <Problem Name> solution`
